Die Query-Ansicht setzt POST /api/v1/run-query ab: Sie liest die Events eines Scopes und filtert sie mit einem CEL-Prädikat über die Variable event. Volle Referenz in der API-Doku.

Die Variable event

event.id
global monotone ID (String, z. B. "42")
event.type
Event-Typ, z. B. "order-placed"
event.subject
Stream-Pfad, z. B. "/orders/42"
event.source
Herkunft (URI-artig)
event.time
Zeitstempel als String (RFC 3339) — für Zeitvergleiche timestamp(event.time)
event.data
die Nutzdaten (dynamisch). Felder defensiv mit has(event.data.x) prüfen, bevor du sie verwendest.

Operatoren & Funktionen (Standard-CEL)

Vergleich / Logik
== != < <= > >= && || ! · a in [..] · cond ? x : y
Strings
s.startsWith(p) · s.endsWith(p) · s.contains(p) · s.matches(re) · size(s)
Existenz / Makros
has(event.data.x) · l.exists(x, p) · l.all(x, p) · l.filter(x, p) · l.map(x, e)
Konvertierung
int(x) double(x) string(x) bool(x) · timestamp(s) duration(s) · type(x)

Das Prädikat muss zu bool evaluieren. Ein leeres Prädikat liefert alle Events im Scope.

Scope & Projektion

Subject
Pflicht, beginnt mit /. Mit rekursiv alles darunter.
lowerBound / upperBound
inklusive Event-ID-Grenzen, schränken den gelesenen Bereich ein.
Limit
max. Treffer (0 = unbegrenzt).
select
punktseparierte Feldpfade (z. B. id, data.amount) → nur diese Felder pro Zeile, Verschachtelung bleibt erhalten.

Verlauf, Favoriten & Export

Verlauf
jede erfolgreiche Abfrage wird gemerkt (im Browser, localStorage) — über Verlauf & Favoriten einsehbar; Klick lädt sie zurück.
Favoriten
mit ★ Favorit die aktuelle Abfrage benannt speichern; bleibt über Sitzungen erhalten.
Export
das Ergebnis als NDJSON (ein Objekt je Zeile) oder CSV (verschachtelte Felder als punktierte Spalten) herunterladen.
